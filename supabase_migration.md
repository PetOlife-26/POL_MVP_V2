# PetoLife — Complete Supabase SQL Migration

Run all the SQL blocks below in the **Supabase SQL Editor** (Dashboard > SQL Editor > New Query).

---

## Step 1: Drop Existing Tables (Safety Guard)

Run this first to clean up any previous schema. This uses `CASCADE` to handle foreign key dependencies.

```sql
-- Safety: Drop tables if they exist (in reverse dependency order)
DROP TABLE IF EXISTS daily_streaks CASCADE;
DROP TABLE IF EXISTS daily_task_logs CASCADE;
DROP TABLE IF EXISTS care_team CASCADE;
DROP TABLE IF EXISTS pet_ids CASCADE;
DROP TABLE IF EXISTS pet_profiles CASCADE;
```

---

## Step 2: Create `pet_profiles` Table

This is the core table. Each row is one pet. The `user_id` column links the pet to the Supabase Auth user who created it.

```sql
CREATE TABLE pet_profiles (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id TEXT,                              -- links to auth.users.id
  petolife_id TEXT NOT NULL UNIQUE,          -- generated PetOLife ID
  pet_type TEXT NOT NULL,                    -- Dog, Cat, Bird, Rabbit, etc.
  pet_name TEXT NOT NULL,
  breed TEXT,
  gender TEXT,
  birth_date DATE,
  weight NUMERIC,
  color TEXT,
  blood_group TEXT,
  identification_marks TEXT,
  pet_photo_url TEXT,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

-- Index for fast user-based lookups
CREATE INDEX idx_pet_profiles_user_id ON pet_profiles(user_id);
```

---

## Step 3: Create `pet_ids` Table

External IDs (KCI, Microchip, etc.) linked to a pet profile.

```sql
CREATE TABLE pet_ids (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  pet_profile_id UUID NOT NULL REFERENCES pet_profiles(id) ON DELETE CASCADE,
  id_name TEXT NOT NULL,
  id_number TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE INDEX idx_pet_ids_profile ON pet_ids(pet_profile_id);
```

---

## Step 4: Create `care_team` Table

Veterinary and emergency contact details per pet.

```sql
CREATE TABLE care_team (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  pet_profile_id UUID NOT NULL REFERENCES pet_profiles(id) ON DELETE CASCADE UNIQUE,
  clinic_name TEXT,
  vet_name TEXT,
  vet_contact TEXT,
  emergency_contact_name TEXT NOT NULL,
  emergency_relationship TEXT,
  emergency_contact_number TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

---

## Step 5: Create `daily_task_logs` Table

Tracks each individual care task per pet per day. Auto-generated by the backend when the user opens the Home screen for the first time that day.

```sql
CREATE TABLE daily_task_logs (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  pet_profile_id UUID NOT NULL REFERENCES pet_profiles(id) ON DELETE CASCADE,
  task_date DATE NOT NULL DEFAULT CURRENT_DATE,
  task_id TEXT NOT NULL,
  task_title TEXT NOT NULL,
  task_icon TEXT,
  task_question TEXT,
  confirm_text TEXT,
  skip_text TEXT,
  suggested_time TEXT,
  status TEXT NOT NULL DEFAULT 'pending',    -- 'pending' | 'completed' | 'skipped'
  completed_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE(pet_profile_id, task_date, task_id)
);

CREATE INDEX idx_task_logs_pet_date ON daily_task_logs(pet_profile_id, task_date);
```

---

## Step 6: Create `daily_streaks` Table

Tracks care streak and cumulative care points per pet.

```sql
CREATE TABLE daily_streaks (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  pet_profile_id UUID NOT NULL REFERENCES pet_profiles(id) ON DELETE CASCADE UNIQUE,
  current_streak INT DEFAULT 0,
  longest_streak INT DEFAULT 0,
  total_care_points INT DEFAULT 0,
  last_completed_date DATE,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

## Complete Schema Overview

| Table | Purpose | Key Relationships |
|-------|---------|-------------------|
| `pet_profiles` | Core pet data | `user_id` -> auth user |
| `pet_ids` | External IDs (KCI, microchip) | `pet_profile_id` -> pet_profiles |
| `care_team` | Vet + emergency contacts | `pet_profile_id` -> pet_profiles (1:1) |
| `daily_task_logs` | Daily care tasks per pet | `pet_profile_id` -> pet_profiles |
| `daily_streaks` | Streak tracking per pet | `pet_profile_id` -> pet_profiles (1:1) |

### Entity Relationship

```
auth.users (Supabase Auth)
    |
    | user_id (TEXT)
    |
    v
pet_profiles ──────┬────── pet_ids (1:many)
                   |
                   ├────── care_team (1:1)
                   |
                   ├────── daily_task_logs (1:many, per day)
                   |
                   └────── daily_streaks (1:1)
```
